# Task 4.2.2: Define CallbackDispatcher Infrastructure

## Objective
Create the core interfaces and base implementation for the multi-protocol callback engine.

## Requirements
- Define `CallbackDispatcher` interface.
- Implement `DefaultCallbackDispatcher` that routes tasks to protocol-specific handlers.
- Integrate `ClientStore` to look up `CallbackConfig` for each task.
- Provide a mechanism for global connection pooling (where applicable).

## Proposed Changes
### `boomerang-core/src/main/java/io/boomerang/server/callback/CallbackDispatcher.java`
- `void dispatch(TimerTask task) throws Exception;`

### `boomerang-core/src/main/java/io/boomerang/server/callback/DefaultCallbackDispatcher.java`
- Implementation that uses `ClientStore` to find the protocol and endpoint.
- Routes to specialized handlers (TCP, HTTP, UDP, gRPC).

## Verification Plan
- Unit test `DefaultCallbackDispatcher` with a mocked `ClientStore` and protocol handlers.
- Ensure proper error handling when a client or protocol is unknown.
