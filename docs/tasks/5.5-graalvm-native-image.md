# Task 5.5: GraalVM Native Image Configuration

This task involves configuring the `boomerang-cli` module to produce a standalone native binary using GraalVM Native Image.

## Objectives
- [x] Ensure `boomerang-cli/build.gradle.kts` is correctly configured for Native Image.
- [x] Resolve any reflection or resource issues related to picocli and JLine.
- [x] Address Netty-specific Native Image requirements.
- [x] Verify the build produces a functional `boomtool` binary.
- [x] Add documentation on how to build the native image.

## Implementation Plan

### 1. Refine Gradle Configuration
- [x] Verify `org.graalvm.buildtools.native` plugin version.
- [x] Ensure `picocli-codegen` is generating the required reflection metadata.
- [x] Configure `graalvmNative` block with necessary build arguments (e.g., `--no-fallback`, `--initialize-at-build-time`).

### 2. Handle Dependencies
- [x] **Netty:** Netty often requires specific initialization flags or substitution files for Native Image. Check if `netty-all` provides these or if manual config is needed.
- [x] **JLine:** Ensure JLine's native hooks (TerminalBuilder) work correctly in the native binary.

### 3. Verification
- [x] Run `./gradlew :boomerang-cli:nativeCompile`.
- [x] Test the generated `boomtool` binary in both one-shot and interactive modes.

### 4. Documentation
- [x] Update `README.md` or create a `docs/BUILD_NATIVE.md` with instructions.

## Success Criteria
- [x] `./gradlew :boomerang-cli:nativeCompile` completes successfully.
- [x] The resulting `boomtool` binary can connect to a running `BoomerangServer`.
- [x] Interactive mode (JLine) works as expected in the native binary.
